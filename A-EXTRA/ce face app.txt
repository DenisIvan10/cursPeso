La examen poti explica aplicatia asa:

Ideea Aplicatiei

Aplicatia prognozeaza cursul MXN/RON publicat de BNR. Ea incarca date istorice din CSV, verifica ultima valoare cu feed-ul oficial BNR XML, antreneaza mai multe modele de machine learning/statistice si afiseaza rezultatul intr-un dashboard web.

Arhitectura

Aplicatia are doua parti:

Backend FastAPI

ruleaza pe http://127.0.0.1:7772
citeste datele MXN din CSV
valideaza datele cu BNR XML
antreneaza modelele
expune endpoint-uri API
gestioneaza chatbot-ul si tool-urile lui
Frontend React

ruleaza pe http://127.0.0.1:5173
afiseaza dashboard-ul
afiseaza cursul curent, prognoza, comparatia modelelor si graficul Plotly
include chatbot-ul in coltul dreapta jos
Fluxul Datelor

Aplicatia citeste fisierul CSV cu coloanele:

Date
Value
Change
Percent
Backend-ul normalizeaza datele in format intern:

date
value
change
percent
Datele sunt sortate cronologic.

Ultimele 14 observatii sunt folosite ca test set.

Restul datelor sunt folosite pentru antrenare.

Ultima valoare din CSV este verificata cu feed-ul oficial BNR:

https://www.bnr.ro/nbrfxrates.xml
Modelele Folosite

Aplicatia compara 3 abordari:

Prophet

model specializat pentru time series
poate modela sezonalitate
necesita CmdStan pentru rulare completa
XGBoost Regressor

model de machine learning pe feature-uri
foloseste lag-uri si medii mobile
in versiunea curenta este modelul castigator
ARIMA / SARIMA

model statistic clasic pentru serii de timp
foloseste autocorelarea seriei
Feature Engineering

Pentru predictie se genereaza variabile precum:

valoarea din ziua anterioara: t-1
valoarea de acum doua zile: t-2
valoarea de acum trei zile: t-3
ziua saptamanii
luna
media mobila pe 7 zile
media mobila pe 14 zile
Evaluarea Modelelor

Modelele sunt evaluate pe acelasi test set de 14 zile folosind:

MAE: eroarea medie absoluta
RMSE: penalizeaza erorile mai mari
MAPE: eroarea procentuala medie
Modelul cu cele mai bune rezultate este ales castigator si este salvat local.

Dashboard

In frontend se vad:

ultimul curs MXN
ultima prognoza
ultimul model castigator
tabel cu metricile modelelor
grafic Plotly cu:
cursul real
prognoza
intervalul de incredere
Chatbot

Chatbot-ul este integrat in coltul dreapta jos si poate folosi functiile aplicatiei ca tool-uri.

Exemple:

daca intrebi „What is the latest MXN rate?”, chatbot-ul apeleaza tool-ul get_rates
daca intrebi de prognoza, apeleaza get_latest_forecast
daca intrebi de comparatia modelelor, apeleaza compare_models
daca ceri actualizarea datelor, apeleaza scrape_bnr_data
daca ceri antrenarea modelelor, apeleaza train_models
Important: tool-urile sunt executate in backend, nu direct in frontend. Asta e mai sigur, pentru ca API keys si logica aplicatiei raman pe server.

Ce Poti Spune Ca Demo

Pornesc backend-ul FastAPI.
Pornesc frontend-ul React.
Aplicatia incarca datele MXN din CSV.
Verific ultima valoare cu XML-ul oficial BNR.
Apas „Train models”.
Aplicatia antreneaza modelele si alege castigatorul dupa MAE/RMSE.
Dashboard-ul afiseaza prognoza si graficul.
Folosesc chatbot-ul pentru a cere cursul sau prognoza.
Limitare de Mentionat

Prophet este instalat, dar pentru rulare completa pe Windows are nevoie de CmdStan. Aplicatia are fallback ca sa nu se blocheze, iar XGBoost si ARIMA/SARIMA ruleaza deja.
